# Optimizing & Configuring

The Publisher sidebar gives you full control over how your model looks and performs. This guide explains every setting.

---

## Quality presets

Quality presets apply a combination of mesh simplification and texture compression to reduce file size. Choose the preset that matches your use case:

| Preset | Texture resolution | Texture quality | Use when |
|---|---|---|---|
| **Base** | 2K WebP | 90% | High-fidelity previews and final renders |
| **Balanced** | 1K WebP | 80% | Recommended for most web experiences |
| **Performance** | 512px WebP | 70% | Smallest file size, mobile and low-end hardware |

All three presets apply mesh simplification, texture optimization, and vertex quantization; the **Performance** preset additionally removes duplicate data.

Preset changes update the current optimized output for the scene. Until scene version history ships, treat optimization changes as mutable scene edits.

---

## Mesh simplification

Under the hood, mesh simplification uses the [glTF-Transform](https://gltf-transform.dev) `simplify` function. The **ratio** controls the fraction of triangles retained (1.0 = no simplification, 0.5 = 50% reduction).

You can override the preset ratio manually in the advanced settings panel.

---

## Texture compression

Textures are re-encoded in the browser using `OffscreenCanvas` (hardware-accelerated where available). The target format is WebP by default (JPEG and PNG are also supported), and the **quality** value maps to the encoder quality in the `0`-`1` range.

> Texture compression runs in the browser with no server round-trip. The viewport preview uses the uncompressed original while compression runs.

## Lighting & environment

### Environment preset

Choose from a set of HDR environment presets that determine ambient lighting, reflections, and image-based lighting (IBL). Presets are grouped into four categories: **Studio** (`studio-key`, `studio-natural`, `studio-soft`), **Nature** (`nature-moonlit`, `nature-park`, `nature-park-overcast`, `nature-snow`), **Night** (`night-building`, `night-city`, `night-pure-sky`, `night-stars`), and **Outdoor** (`outdoor-golden-hour`, `outdoor-noon`, `outdoor-overcast`, `outdoor-sky`).

### Stage settings

The `Stage` component from [@react-three/drei](https://github.com/pmndrs/drei) automatically positions the model and adjusts the camera to fit. Use **adjustCamera** to control how tightly the camera frames the model (higher = tighter crop).

### Background color

Override the transparent background with a solid color using the background color picker in the environment section.

---

## Camera

### Initial position

Set the camera's starting world-space position (`x, y, z`). The camera will animate to this position when the scene loads.

### Field of view (FoV)

Adjust the perspective camera's FoV in degrees. Wider FoV (e.g. 90°) exaggerates depth; narrower (e.g. 30°) produces a more orthographic feel.

### Auto-rotate

Enable auto-rotate to continuously orbit the model. Configure the speed and the pause delay after user interaction (`controlsTimeout` in milliseconds).

---

## Grid

Grid controls are currently not active in the viewer render flow. If enabled in a future release, this section will include supported options and examples.

---

## Saving settings

Settings are saved with the scene when you click **Save**. They are restored every time you or a viewer loads the scene embed.
